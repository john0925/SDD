# Capability Spec: Meal Catalog

## 1. 概述
提供「健康日常」品牌的核心餐點展示功能，著重於精確的營養標示與編輯感的視覺呈現。

## 2. 核心功能
### 2.1 餐點列表展示
- 展示所有可用餐點。
- 每個餐點需包含：名稱、描述、高品質圖片、熱量、三大營養素 (蛋白質、碳水、脂肪)。
- 視覺需遵循「數位策展人」風格：無邊框設計、階層式背景色調。

### 2.2 營養需求篩選 (Nutrient Filtering)
- 使用者可依據「碳水攝取需求」篩選餐點：
  - **全部**：顯示所有餐點。
  - **無碳 (No Carb)**：適合極低碳飲食者。
  - **低碳 (Low Carb)**：適合日常控醣者。
  - **高碳 (High Carb)**：適合運動後能量補充者。

### 2.3 精確營養數據標示
- 熱量需以 `kcal` 為單位。
- 蛋白質、碳水、脂肪需以 `g` 為單位。
- **視覺規範**：數據標籤需具備臨床信任感 (使用 Inter 字體)，並與「加入購物車」等行動呼籲按鈕有明確的視覺區隔。

## 3. 資料模型 (Meal Model)
| 屬性 | 說明 | 範例 |
| :--- | :--- | :--- |
| `ID` | 唯一識別碼 | `meal-001` |
| `Name` | 餐點名稱 | `嫩煎雞胸餐盒` |
| `CarbCategory` | 碳水分類 | `none`, `low`, `high` |
| `Calories` | 總熱量 | `450` |
| `Protein` | 蛋白質 (g) | `45` |
| `Carbs` | 碳水 (g) | `30` |
| `Fat` | 脂肪 (g) | `15` |
| `Price` | 價格 | `180` |
