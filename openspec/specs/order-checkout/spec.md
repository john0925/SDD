# Capability Spec: Order Checkout

## 1. 概述
處理「健康日常」的結帳流程，包含餐點清單確認、取餐時間設定與訂單摘要計算。

## 2. 功能需求

### Requirement: 購物車狀態管理
系統 **SHALL** 提供一個全域狀態來儲存使用者選購的餐點，並支援跨頁面存取。

#### Scenario: 加入餐點至購物車
- **WHEN** 使用者在菜單頁面點擊餐點的「加入購物車」按鈕
- **THEN** 系統將該餐點加入購物車清單，若已存在則增加數量，並即時更新導覽列的購物車圖示

### Requirement: 結帳資訊聚合計算
系統 **SHALL** 在結帳頁面以「托盤」為單位展示餐點清單，並即時計算每個托盤的營養數據、**托盤金額小計**與全訂單的總計金額。**系統還必須顯示本次消費預計獲得的會員點數。**

#### Scenario: 托盤金額小計與預計點數
- **WHEN** 使用者進入結帳頁面
- **THEN** 系統顯示每個托盤的金額小計，並在頁面底部顯示總金額與「本次預計獲得 X 點」

### Requirement: 取餐時間預設邏輯
系統在結帳頁面 **MUST** 提供預設的取餐時間。

#### Scenario: 自動產生預設取餐時間
- **WHEN** 使用者進入結帳頁面
- **THEN** 系統自動設定取餐時間為當前時間加上 30 分鐘，並允許使用者手動修改

### Requirement: 訂單成功視覺回饋
系統在訂單提交成功後，**MUST** 提供活潑且具備配色風格的視覺回饋，展示積點進度。

#### Scenario: 展現集點成功動畫
- **WHEN** 使用者成功提交訂單
- **THEN** 系統導向成功頁面，並顯示動態的「印章卡」動畫，逐點填滿圓圈並展現 bouncy 效果
