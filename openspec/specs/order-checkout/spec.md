# Capability Spec: Order Checkout

## 1. 概述
處理「健康日常」的結帳流程，包含餐點清單確認、取餐時間設定與訂單摘要計算。

## 2. 功能需求

### Requirement: 購物車狀態管理
系統 **SHALL** 提供一個全域狀態來儲存使用者選購的餐點，並支援跨頁面存取。

#### Scenario: 加入餐點至購物車
- **WHEN** 使用者在菜單頁面點擊餐點的「加入購物車」按鈕
- **THEN** 系統將該餐點加入購物車清單，若已存在則增加數量，並即時更新導覽列的購物車圖示

### Requirement: 結帳資訊聚合計算
系統 **SHALL** 在結帳頁面以「托盤」為單位展示餐點清單，並即時計算每個托盤的營養數據與全訂單的總計金額。

#### Scenario: 托盤式結帳展示
- **WHEN** 使用者進入結帳頁面
- **THEN** 系統將餐點按托盤分組顯示，每個托盤旁邊/下方展示該托盤的獨立營養分析，頁面底部展示整筆訂單的總計金額

### Requirement: 取餐時間預設邏輯
系統在結帳頁面 **MUST** 提供預設的取餐時間。

#### Scenario: 自動產生預設取餐時間
- **WHEN** 使用者進入結帳頁面
- **THEN** 系統自動設定取餐時間為當前時間加上 30 分鐘，並允許使用者手動修改
