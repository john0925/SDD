# Capability Spec: Smart Nutrition Analysis

## 1. 概述
實作精準營養評分引擎，根據餐點成分、熱量與多樣性提供即時的飲食回饋。

## 2. 功能需求

### Requirement: 營養均衡度評分引擎
系統 **MUST** 實作一套基於「三大營養素比例」、「熱量達成度」與「食材多樣性」的綜合評分演算法。

#### Scenario: 三大營養素佔比評分 (60%)
- **WHEN** 訂單建立後
- **THEN** 系統以歐幾里得距離公式計算實際比例與目標值（P:30%, C:40%, F:30%）的偏差，得出佔比得分

#### Scenario: 熱量目標達成度評分 (30%)
- **WHEN** 訂單建立後
- **THEN** 若總熱量落在 500-800 kcal 區間則得滿分，每偏離 50 kcal 扣 5 分

#### Scenario: 食材多樣性獎勵評分 (10%)
- **WHEN** 訂單建立後
- **THEN** 若餐點包含「非精製澱粉」標籤加 5 分，包含「多樣蔬菜」標籤加 5 分

### Requirement: 營養夥伴總結展示
系統 **SHALL** 在訂單成功頁面展示最終的評分結果與視覺化反饋。

#### Scenario: 展示飲食均衡度百分比
- **WHEN** 使用者進入訂單成功頁面
- **THEN** 系統展示計算後的均衡度總分，並以進度條與評語（如「優秀」）呈現
